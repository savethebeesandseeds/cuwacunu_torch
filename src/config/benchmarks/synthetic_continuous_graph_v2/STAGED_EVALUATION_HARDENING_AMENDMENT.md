# Staged-evaluation hardening amendment

Status: fixed on 2026-07-16 after the physical-source exposure invalidated the
first canonical representation and MDN runs, and before any clean isolated
source, clean checkpoint, frozen-feature capture, affine validation report,
conditional route, or certified-development feature row existed. It changes
no scientific candidate or selection rule.

The first staged capture and affine implementations are non-executable
prototypes. Their development/certification separation is retained, but the
clean isolated-source replacements must also satisfy the controls below.

## Pre-certified selection lock

Development fitting and validation selection produce an immutable report for
each trained feature arm. Before a certified probe is opened or parsed, the
certified evaluator must read the corresponding development report and verify
that the newly reconstructed train-only solve has exactly the same declared
candidate validity, selected candidate index, ridge, solve diagnostics, and
validation metrics. Any difference fails before certified input is opened.

The selected model may be deterministically reconstructed from the unchanged
train probe and selected ridge; this is not a statistical refit and may not
include validation or certified rows. The certified report records that its
selection lock was verified. Runner-side comparison after certified scoring
is retained as a redundant check, not the first enforcement point.

## Exact trigger provenance

The clean capture runner defines, rather than accepts from the trigger, the
only valid affine development runtime, evaluator binary, development status,
three validation reports, and master manifest paths. It requires exact path
equality, regular non-symlinked files, immutable modes, verified hashes, and a
valid master manifest. The development status must bind those exact reports,
the capture development receipt, isolated-source closure, evaluator source,
runner source, binary, and both source-isolation documents.

The trigger must also bind and match every representation diagnostic,
conditional-certification, source-isolation, and staged-hardening document.
Only the trained raw-96 report supplies route metrics. A self-consistent
trigger pointing to substitute artifacts is not authorization.

## Complete attempt verification

The clean certified-attempt verifier checks every field written by its
publisher, including schema, immutable mode, attempt ordinal, exact range and
job path, route, source closure, capture development, trigger, config,
representation and MDN checkpoints, evaluator binary and sources, all
amendment hashes, and final/policy access flags. An existing attempt without a
complete valid job remains consumed and cannot be retried.

All clean jobs additionally require the isolated 3,264-anchor source domain
and reject canonical `data/raw`. No V2 operation is independent final
evidence.
